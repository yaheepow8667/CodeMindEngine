# 业务域划分与包路径说明

## 1. 概述
本文档对系统中的20个核心业务表进行业务域划分，并为每个业务域分配对应的英文包路径，用于代码生成时的模块组织。

## 2. 业务域划分

### 2.1 团队与协作管理域
**核心功能**：管理团队、成员和邀请的协作功能
**英文包路径**：`module_team_collaboration`
**包含表**：
- teams - 团队表
- team_members - 团队成员表
- invitations - 邀请表

### 2.2 项目与蓝图管理域
**核心功能**：管理项目生命周期和蓝图设计
**英文包路径**：`module_project_blueprint`
**包含表**：
- projects - 项目表
- blueprints - 蓝图表
- blueprint_changes - 蓝图变更表

### 2.3 代码生成与部署域
**核心功能**：管理代码生成流程和部署配置
**英文包路径**：`module_code_generation`
**包含表**：
- generation_jobs - 生成任务表
- generated_artifacts - 生成的工件表
- deployment_configs - 部署配置表

### 2.4 模板与插件管理域
**核心功能**：管理代码生成模板和扩展插件
**英文包路径**：`template_plugin`
**包含表**：
- templates - 模板表
- template_versions - 模板版本表
- plugins - 插件表

### 2.5 系统资源与安全域
**核心功能**：管理系统资源和安全控制
**英文包路径**：`system_resource`
**包含表**：
- api_tokens - API令牌表
- resources - 资源表
- subscriptions - 订阅表

### 2.6 日志与审计域
**核心功能**：记录系统操作和审计信息
**英文包路径**：`log_audit`
**包含表**：
- system_logs - 系统日志表
- api_logs - API日志表
- audit_logs - 审计日志表

### 2.7 通知与消息域
**核心功能**：处理系统通知和用户消息
**英文包路径**：`notification_message`
**包含表**：
- notifications - 通知表
- messages - 消息表

## 3. 使用建议

### 3.1 代码组织结构
生成代码后，建议按照以下结构组织模块：

```
ruoyi-fastapi-backend/
├── module_team_collaboration/     # 团队与协作管理域
├── module_project_blueprint/      # 项目与蓝图管理域
├── module_code_generation/        # 代码生成与部署域
├── module_template_plugin/        # 模板与插件管理域
├── module_system_resource/        # 系统资源与安全域
├── module_log_audit/              # 日志与审计域
└── module_notification_message/   # 通知与消息域
```

### 3.2 包路径命名规范
1. 所有包路径采用小写字母
2. 使用下划线分隔单词
3. 保持名称简洁明了，反映业务功能

### 3.3 模块名命名规范
- 与表名保持一致的单数形式（如team、project等）
- 遵循Python命名规范（小写字母+下划线）
